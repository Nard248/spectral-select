<task type="auto">
  <name>Task 1: Add SpectraData.to_pickle() method</name>
  <files>spectral_select/types.py</files>
  <action>
Add a to_pickle() instance method to SpectraData for saving processed data.

Signature:
```python
def to_pickle(self, path: Union[str, Path]) -> Path:
    """Save to pickle format compatible with from_pickle().

    Args:
        path: Output file path. Parent directories created if needed.

    Returns:
        The path the file was written to.
    """
```

Implementation:
1. Convert Path if string provided
2. Create parent directories if they don't exist (mkdir parents=True, exist_ok=True)
3. Build output dict in format expected by from_pickle():
   - 'data': {ex_nm_str: {'cube': array, 'wavelengths': list}}
   - 'excitation_wavelengths': list of floats
   - 'mask': numpy array or None
   - 'exposure_times': {ex_nm_str: float} if available
   - 'laser_powers': {ex_nm_str: float} if available
4. Write using pickle.dump with protocol=pickle.HIGHEST_PROTOCOL
5. Return the path

Note: The project uses pickle because JSON can't efficiently handle large numpy arrays. The security warning applies to loading untrusted pickle content - saving our own data is safe. from_pickle() already exists and is used throughout the codebase.

Do NOT:
- Change the existing from_pickle format expectations
- Add compression (keep it simple)
